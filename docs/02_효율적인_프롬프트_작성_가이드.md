## 토큰이란?

**토큰** = AI가 읽고 쓰는 텍스트의 단위

- 영어: 단어 1개 ≈ 1~2 토큰
- 한글: 글자 1개 ≈ 2~3 토큰
- 코드: 한 줄 ≈ 5~20 토큰

**왜 중요해요?**
- Claude는 토큰 사용량에 따라 비용이 발생
- 토큰을 아끼면 같은 비용으로 더 많이 사용 가능
- 불필요한 대화 왕복을 줄이면 시간도 절약

## 효율적인 프롬프트의 원칙

### 1. 한 번에 구체적으로 요청하기

**나쁜 예:**
```
웹사이트 만들어줘
→ 어떤 웹사이트요?
→ 투어 정보 사이트요
→ 어떤 정보를 넣을까요?
→ 날짜랑 장소요
... (토큰 낭비!)
```

**좋은 예:**
```
투어 정보 웹사이트를 만들어줘.
- 밴드명: 솔루션스
- 투어명: 2026 전국투어
- 필요한 기능: 공연 일정, 지도, 셋리스트
- 디자인: 다크 테마, 심플한 느낌
```

### 2. 참고 자료 제공하기

```
이런 느낌으로 만들어줘:
- 참고 사이트: [URL]
- 색상: #1a1a2e (배경), #ff6b6b (강조색)
- 폰트: 고딕 계열
```

### 3. 기술 스택 명시하기

```
React + TypeScript + Vite로 만들어줘.
스타일은 CSS modules 사용.
```

### 4. 우선순위 정하기

```
1순위: 일정 표시 기능 (필수)
2순위: 지도 연동 (있으면 좋음)
3순위: SNS 공유 (나중에 추가)
```

## 웹사이트 요청 프롬프트 템플릿

### 기본 템플릿

```markdown
## 프로젝트 개요
- 프로젝트명: [이름]
- 목적: [한 줄 설명]
- 타겟 사용자: [누가 사용하는지]

## 기술 스택
- 프레임워크: React + TypeScript + Vite
- 스타일링: CSS (또는 Tailwind/styled-components)
- 배포: Vercel

## 필요한 페이지/기능
1. [기능1]: [상세 설명]
2. [기능2]: [상세 설명]
3. [기능3]: [상세 설명]

## 디자인 요구사항
- 테마: [다크/라이트]
- 메인 색상: [색상코드]
- 강조 색상: [색상코드]
- 분위기: [키워드들]

## 데이터 구조 (있으면)
- [데이터1]: [형식/예시]
- [데이터2]: [형식/예시]

## 참고 사항
- [추가 요구사항]
- [주의할 점]
```

### 실제 사용 예시 (투어 웹사이트)

```markdown
## 프로젝트 개요
- 프로젝트명: 솔루션스 2026 투어 페이지
- 목적: 밴드 전국투어 일정 및 정보 제공
- 타겟 사용자: 팬덤

## 기술 스택
- React + TypeScript + Vite
- CSS modules
- Vercel 배포

## 필요한 기능
1. 공연 일정: 날짜, 장소, 공연장 정보 표시
2. 지도: 각 공연장 위치를 한국 지도에 마커로 표시
3. 셋리스트: Google Sheets 연동으로 실시간 업데이트
4. 전광판: 팬들이 메시지 남기는 기능

## 디자인
- 다크 테마 (배경: #0a0a0a)
- 강조색: 네온 계열 (#ff6b6b, #4ecdc4)
- 분위기: 공연장, 밴드, 에너지틱

## 데이터
- 공연 데이터: JSON 하드코딩 후 시트 연동 확장
- 구조: { city, date, venue, coordinates }
```

## 상황별 프롬프트 예시

### 1. 새 기능 추가 요청

```
[기능명] 기능을 추가해줘.

요구사항:
- [기능 설명]
- [동작 방식]
- [UI 위치]

참고:
- 기존 [컴포넌트명]과 비슷한 스타일로
- [관련 파일] 수정 필요할 수 있음
```

**예시:**
```
셋리스트를 Google Sheets로 관리하는 기능 추가해줘.

요구사항:
- 시트 컬럼: A=춘천, B=전주, C=부산, D=대구, E=대전, F=서울
- 각 컬럼에 곡 목록이 세로로 입력됨
- 첫 행은 헤더 (도시명)

시트 ID: 1nxs4vtE8nkZbMNlkuOZH62hZSxHqZxf4R-DWMdG1uDk
```

### 2. 버그 수정 요청

```
[문제 상황] 버그 수정해줘.

현상:
- [어떤 문제가 발생하는지]
- [언제/어디서 발생하는지]

예상 원인:
- [추측되는 원인]

관련 파일:
- [파일 경로]
```

**예시:**
```
셋리스트가 3열로 표시되는 버그 수정해줘.

현상:
- 20곡 이상일 때 3열이 됨
- 2열이어야 함 (1열: 1-10곡, 2열: 11곡~)

관련 파일:
- src/components/SetlistDisplay.tsx
- src/App.css
```

### 3. 스타일 수정 요청

```
[대상]의 스타일을 수정해줘.

현재: [현재 상태]
원하는 것: [원하는 상태]

세부사항:
- [구체적인 수치/색상]
```

**예시:**
```
전광판 속도를 조절해줘.

현재: 너무 빠름 (20초)
원하는 것: 메시지 개수에 따라 동적 조절

세부사항:
- 메시지당 8초
- 최소 10초
```

### 4. 외부 서비스 연동 요청

```
[서비스명] 연동해줘.

서비스 정보:
- URL/ID: [정보]
- 인증 방식: [공개/API키 등]

데이터 형식:
- [어떤 데이터를 가져올지]
- [데이터 구조]

사용 위치:
- [어디에 표시할지]
```

## 추가하면 좋은 스킬 문서

프로젝트 루트에 `CLAUDE.md` 또는 `.claude/` 폴더를 만들어 다음 정보를 추가하면 AI가 프로젝트를 더 잘 이해해요.

### CLAUDE.md 템플릿

```markdown
# 프로젝트 컨텍스트

## 개요
이 프로젝트는 [설명]입니다.

## 기술 스택
- Frontend: React 18 + TypeScript
- Build: Vite
- 스타일: CSS modules
- 배포: Vercel

## 폴더 구조
```
src/
├── components/   # UI 컴포넌트
├── data/         # API 및 데이터 로직
├── types/        # TypeScript 타입 정의
└── App.tsx       # 메인 앱
```

## 주요 파일
- `src/data/tourData.ts`: 투어 일정 데이터
- `src/data/setlistApi.ts`: 셋리스트 API
- `src/data/marqueeApi.ts`: 전광판 API

## 환경변수
- `VITE_GOOGLE_SHEETS_ID`: 투표 시트
- `VITE_SETLIST_SHEET_ID`: 셋리스트 시트
- `VITE_MARQUEE_SHEET_ID`: 전광판 시트

## 코딩 컨벤션
- 컴포넌트: PascalCase (예: TourMap.tsx)
- 함수: camelCase
- 상수: UPPER_SNAKE_CASE
- 한글 주석 사용 OK

## 자주 사용하는 명령어
- `npm run dev`: 개발 서버 실행
- `npm run build`: 프로덕션 빌드
- `git push`: Vercel 자동 배포

## 주의사항
- .env 파일은 커밋하지 않음
- Vercel 환경변수는 별도 설정 필요
```

### 디자인 시스템 문서 (선택)

```markdown
# 디자인 시스템

## 색상
- 배경: #0a0a0a
- 텍스트: #ffffff
- 서브텍스트: #888888
- 강조1: #ff6b6b (레드 네온)
- 강조2: #4ecdc4 (민트 네온)
- 보더: #333333

## 타이포그래피
- 제목: 1.5rem, bold
- 본문: 1rem, regular
- 캡션: 0.875rem, light

## 간격
- 섹션: 2rem
- 요소: 1rem
- 내부: 0.5rem

## 반응형 브레이크포인트
- 모바일: ~768px
- 태블릿: 769px~1024px
- 데스크탑: 1025px~
```

## 자주 하는 실수와 해결법

### 실수 1: 너무 추상적인 요청

**나쁜 예:** "더 예쁘게 해줘"
**좋은 예:** "버튼 모서리를 8px 둥글게, 호버시 배경색 #ff6b6b로"

### 실수 2: 컨텍스트 없이 수정 요청

**나쁜 예:** "안 돼요"
**좋은 예:** "춘천 클릭했는데 셋리스트가 안 보여요. 콘솔에 에러 없음."

### 실수 3: 여러 기능을 한꺼번에 요청

**나쁜 예:** "로그인, 회원가입, 마이페이지, 프로필 수정, 알림 기능 만들어줘"
**좋은 예:** 기능별로 나눠서 요청, 또는 우선순위 명시

### 실수 4: 환경 정보 누락

**나쁜 예:** "배포가 안 돼요"
**좋은 예:**
```
Vercel 배포 실패
에러: VITE_SETLIST_SHEET_ID is not defined
환경변수 설정 확인 방법?
```

## 효율적인 대화 패턴

### 1. 빌드업 패턴 (큰 기능)

```
1회차: 기본 구조 + 핵심 기능
2회차: 스타일링 + 세부 조정
3회차: 버그 수정 + 최적화
```

### 2. 원샷 패턴 (작은 기능)

```
[기능] 추가해줘.
- 조건1
- 조건2
- 스타일: 기존 [컴포넌트]와 동일
```

### 3. 디버깅 패턴

```
문제: [현상]
시도: [해본 것]
결과: [어떻게 됐는지]
예상: [추측되는 원인]
```

## 토큰 절약 팁

1. **스크린샷 활용**: "이렇게 생겼는데" → 스크린샷 첨부
2. **코드 직접 첨부**: "이 파일 수정해줘" → 파일 경로만 알려주면 Claude가 읽음
3. **반복 요청 줄이기**: 한 번에 구체적으로
4. **CLAUDE.md 활용**: 매번 설명 안 해도 됨
5. **git 커밋 자주**: 롤백 쉬움, 재요청 줄어듦

## 마무리

좋은 프롬프트의 핵심:
1. **구체적으로** - 추측하게 하지 않기
2. **구조적으로** - 목록, 섹션 활용
3. **맥락과 함께** - 왜 필요한지, 어디에 쓰는지
4. **예시와 함께** - 참고 자료, 색상코드 등

이 가이드를 참고해서 효율적으로 AI와 협업해보세요!
